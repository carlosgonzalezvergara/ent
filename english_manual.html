1. System fundamentals

ENT is a digital environment designed to assist in syntactic analysis based on the immediate constituent model. Its goal is to offer an intuitive interface for creating tree diagrams.

ENT is a manual assistant: the user makes all analytical decisions, while the program manages hierarchical visualization, label consistency, and image generation.

2. Local file management

ENT allows you to save and load analysis progress using files with the proprietary .ent extension. These files are JSON data documents that preserve the tree structure, labels, and original expression.

  • Save: Access 📁 Files and select 💾 Save .ent. The file will be downloaded to your computer.
  • Load: Use the 📂 Load .ent option to import a previously saved analysis.

When saving, the filename is automatically generated from the first words of the sentence.

Auto-save

In addition to manually saving .ent files, ENT keeps a local auto-save of your work (both in the main mode and in Exercise mode). If you close the tab or your browser, ENT will offer to restore the last auto-saved state when you reopen the app. This auto-save is stored only in your browser (nothing is uploaded), so it may be lost if you clear the site data.

Zoom

ENT includes zoom controls to help you inspect the diagram comfortably, especially when working with long sentences or deep trees.

  • Use the [−] and [+] buttons in the top-right corner to zoom out/in.
  • Click the percentage to reset to 100%.
  • Zoom range: 40% to 200% in 20% increments.
  • The controls appear only when a diagram is present.

3. Analysis process

Creating the base tree

  1. Type the sentence in the main field.
  2. Click Analyze. The root node is created with the complete sentence.

From that point, the work consists of dividing constituents, assigning labels, and when necessary, introducing extra nodes or marking heads.

Undo and redo

ENT has a history system that allows you to undo and redo actions. In the toolbar, next to the "Analyze" button, you'll find the ↶ (undo) and ↷ (redo) buttons. This allows you to easily correct errors or experiment with different analyses without fear of losing your work. You can also use keyboard shortcuts (see section 9).

Normalization and contraction handling

When starting the analysis, ENT processes the input text to separate contracted forms into their component words, making syntactic division more precise.

Automatic expansion. Unambiguous contractions are expanded automatically: won't → will not, can't → cannot, don't → do not (and all other n't forms), I'm → I am, 'll → will, 've → have, and 're → are.

Ambiguous contractions. The forms 's and 'd are ambiguous: 's can represent is, has, or a possessive marker, while 'd can represent would or had. When ENT detects these forms, it displays a dialog listing each ambiguous word with a dropdown menu so the user can select the correct interpretation before the analysis begins (e.g., he's → he is / he has / possessive).

4. Division mechanics

Dividing a terminal node

To divide a constituent, click on a light blue text box (the node). ENT enters "division mode" and displays cut bars between words so you can select the boundaries between constituents.

  • Select one or more cuts (between words). Each cut creates a new constituent.
  • Click Split to generate the child nodes.
  • Use Cancel if you entered division mode by mistake.

Consolidating to prevent division

If a node has more than one orthographic word but corresponds to a single syntactic word, you can "consolidate" it so that it can no longer be divided. Use the ∪ (orange) button at the center of the bottom edge of the node. The node will turn white and will behave as an absolute terminal node.

To allow division again on a consolidated node, click the ✂ (orange) button that replaces the ∪ button on consolidated nodes.

Expanding a terminal node

On terminal nodes that cannot be divided (either because they contain a single word or because they were consolidated), you will find a ⑃ (orange) button at the center of the bottom edge. This button opens a dialog that allows you to create child constituents manually: you can specify the number of constituents (from 2 to 8) and enter the text for each one. This is useful when you need to subdivide a node into constituents that do not correspond to the words in the original text.

Undoing a division

If you have divided a node into its child nodes but regret it and want to go back, use the ↩ button (Undo split). This removes the node's children.

Deleting node text

If you prefer, you can delete a node's text once you've divided it. To do this, you can use the red ⨉ button you'll find in the upper right corner of the node.

Marking a node as head

In absolute terminal nodes (that is, nodes without children that are no longer divisible), you'll find an H button centered on the top edge of the node. This button is not shown in light blue nodes that can still be divided, but it is available in nodes that have been consolidated with ∪. When activated, the node's text is displayed in bold, making it easier to identify the head of the phrase.

5. The label system

Each node offers two fields: upper label (function) and lower label (category). To edit, click on the label and type the corresponding value.

Accessing suggestions

ENT offers two complementary ways to access the label catalog. The first is the manual selector: pressing the ☰ button next to the label opens a window with the full list, in which you can filter the options through an internal search field. The second is automatic suggestion by typing: if you start typing directly in the label, the catalog unfolds on its own and filters the relevant options live as you type. For example, typing the letter n displays all labels whose abbreviation or full name begin with that letter; adding o restricts the list to those that match no, and so on. In both cases, selecting an option from the list causes ENT to insert the corresponding abbreviation in the label field.

If the node you are editing is located near the bottom of the canvas, the list automatically repositions itself above the node and to one side, so that the node being edited remains visible and the list is not clipped by the edge of the screen.

Split labels (two values)

You can split a label into two parts using the ⇄ button. This is useful when you need two values in the same band (for example, two functions or two categories). ENT preserves the separator and renders both parts.

Formatting within labels

ENT allows you to edit the text formatting of labels:

  • Bold: **text**
  • Italic: *text*
  • Text in subscript: _i_
  • Text in superscript: __n__

6. Extra nodes and elision

Adding extra constituents

ENT allows you to add extra constituents by creating an immediate sibling of an existing node. This option is available only in absolute terminal nodes, including nodes that became terminal through consolidation.

These nodes display two green buttons: ← + on the left edge and + → on the right edge. Clicking either one opens a dialog where you can enter the text for the new constituent.

Upon confirmation, the new sibling is inserted immediately to the left or to the right of the selected node. The new node becomes a child of the same parent as the node from which the button was pressed. This is useful, for example, for representing elided material or adding missing constituents without altering the existing structure.

7. Exercise mode

Exercise mode allows you to analyze a sentence while having a reference model in the background. You work with a "blank" tree and then compare it with the solution.

When entering exercise mode, you will be asked to load a .ent file from your computer or from an example bank.

How to start an exercise from the Example Bank

  1. Open ☁️ Example bank from the files panel or with the corresponding button in exercise mode.
  2. Paste the URL of the Google Drive folder containing the .ent files
  3. Select a .ent file from the bank.
  4. Choose 📚 Load as exercise. ENT creates an empty tree for the student and saves the file's tree as the model.

Comparison with the model

When you finish analyzing the exercise, use Compare. ENT will display two panels: Your analysis, on the left, and Model analysis, on the right. Differences are marked visually: correctly identified constituents and labels appear in green; incorrect ones in red.

At the top of the comparison view you can see the number of correct answers and the achievement percentage in relation to the model.

From that view you can return to editing (← Back to editing) or start another exercise (New exercise).

8. Example bank (Google Drive)

Allows instructors to create a library of exercises accessible from the cloud.

If you don't have your own folder, ENT offers a link to the ENT example bank, a test folder with sample exercises that you can use to familiarize yourself with the system.

Minimum requirements

  • Bank files must be saved as .ent.
  • The Drive folder must be shared as "Anyone with the link" (read permission).
  • In ENT, simply paste the link (or ID) of the folder and connect.

How it is organized and what is shown

  • ENT lists the .ent files that are in the root of the folder.
  • It also checks the direct subfolders (one level) and lists their .ent files.
  • If a subfolder is not accessible or doesn't contain .ent files, it simply doesn't appear in the bank.

10. Exporting results

ENT exports the analysis as a PNG image with a transparent background so you can insert it into documents, slides, or articles.

  • Export diagram: exports the diagram as it appears (with boxes).
  • Export tree diagram: exports a "cleaner" style (technical mode), hiding editing controls and reducing non-essential visual elements.

During export, ENT automatically hides interactive buttons so they don't appear in the final image.
