1. System fundamentals

ENT is a digital environment designed to assist in syntactic analysis based on the immediate constituent model. Its goal is to offer an intuitive interface for creating tree diagrams.

ENT is a manual assistant: the user makes all analytical decisions, while the program manages hierarchical visualization, label consistency, and image generation.

2. Local file management

ENT allows you to save and load analysis progress using files with the proprietary .ent extension. These files are JSON data documents that preserve the tree structure, labels, and original expression.

  • Save: Access üìÅ Files and select üíæ Save .ent. The file will be downloaded to your computer.
  • Load: Use the üìÇ Load .ent option to import a previously saved analysis.

When saving, the filename is automatically generated from the first words of the sentence.

Auto-save

In addition to manually saving .ent files, ENT keeps a local auto-save of your work (both in the main mode and in Exercise mode). If you close the tab or your browser, ENT will offer to restore the last auto-saved state when you reopen the app. This auto-save is stored only in your browser (nothing is uploaded), so it may be lost if you clear the site data.

Zoom

ENT includes zoom controls to help you inspect the diagram comfortably, especially when working with long sentences or deep trees.

  • Use the [‚àí] and [+] buttons in the top-right corner to zoom out/in.
  • Click the percentage to reset to 100%.
  • Zoom range: 40% to 200% in 20% increments.
  • The controls appear only when a diagram is present.

3. Analysis process

Creating the base tree

  1. Type the sentence in the main field.
  2. Click Analyze. The root node is created with the complete sentence.

From that point, the work consists of dividing constituents, assigning labels, and when necessary, introducing extra nodes or marking heads.

Undo and redo

ENT has a history system that allows you to undo and redo actions. In the toolbar, next to the "Analyze" button, you'll find the ‚Ü∂ (undo) and ‚Ü∑ (redo) buttons. This allows you to easily correct errors or experiment with different analyses without fear of losing your work. You can also use keyboard shortcuts (see section 9).

Normalization and contraction handling

When starting the analysis, ENT processes the input text to separate contracted forms into their component words, making syntactic division more precise.

Automatic expansion. Unambiguous contractions are expanded automatically: won't ‚Üí will not, can't ‚Üí cannot, don't ‚Üí do not (and all other n't forms), I'm ‚Üí I am, 'll ‚Üí will, 've ‚Üí have, 're ‚Üí are, and let's ‚Üí let us.

Ambiguous contractions. The forms 's and 'd are ambiguous: 's can represent is, has, or a possessive marker, while 'd can represent would or had. When ENT detects these forms, it displays a dialog listing each ambiguous word with a dropdown menu so the user can select the correct interpretation before the analysis begins (e.g., he's ‚Üí he is / he has / possessive).

4. Division mechanics

Dividing a terminal node

To divide a constituent, click on a light blue text box (the node). ENT enters "division mode" and displays cut bars between words so you can select the boundaries between constituents.

  • Select one or more cuts (between words). Each cut creates a new constituent.
  • Click Split to generate the child nodes.
  • Use Cancel if you entered division mode by mistake.

Consolidating to prevent division

If a node has more than one orthographic word but is only one syntactic word, you can "consolidate" it so it cannot be divided. Use the ‚à™ button you'll find in the upper left corner of the node. The node will turn white and the √∑ button will appear in its upper left corner, in case you want to make it divisible again.

Undoing a division

If you have divided a node into its child nodes but regret it and want to go back, use the ‚Ü© button (Undo split). This removes the node's children.

Expanding a single-word node

If a terminal node has a single word, ENT allows you to "expand" it to add constituents below. This is useful for adding, for example, elided constituents. Press the √∑ button in the upper left corner. This will generate an identical child node and a + button will appear below the node, allowing you to add extra constituents.

Deleting node text

If you prefer, you can delete a node's text once you've divided it. To do this, you can use the red ‚®â button you'll find in the upper right corner of the node.

Marking a node as head

In terminal nodes (those without children), you'll find an H button that allows you to mark that constituent as head. When activated, the node's text will be displayed in bold, making it easier to visually identify the heads of each phrase in the diagram.

5. The label system

Each node offers two fields: upper label (function) and lower label (category). To edit, click on the label and type the corresponding value.

Accessing suggestions

In the labels you can open the suggestion selector by pressing the ‚ò∞ button. This will open a window with a list where you can search for your preferred labels. When choosing a label, ENT inserts its abbreviation in the corresponding space.

Split labels (two values)

You can split a label into two parts using the ‚áÑ button. This is useful when you need two values in the same band (for example, two functions or two categories). ENT preserves the separator and renders both parts.

Formatting within labels

ENT allows you to edit the text formatting of labels:

  • Bold: **text**
  • Italic: *text*
  • Text in subscript: _i_
  • Text in superscript: ¬®n¬®

6. Extra nodes and elision

Adding an "extra" child node

When a node has already been divided, a + button will appear at the bottom, allowing you to insert a new child with free text (for example, for elided constituents).

When creating an extra node, ENT will ask you for the text and then the position: At the beginning, At the end, or Between "X" and "Y". This way you can decide exactly where it is inserted among the node's children.

7. Exercise mode

Exercise mode allows you to analyze a sentence while having a reference model in the background. You work with a "blank" tree and then compare it with the solution.

When entering exercise mode, you will be asked to load a .ent file from your computer or from an example bank.

How to start an exercise from the Example Bank

  1. Open ‚òÅÔ∏è Example bank from the files panel or with the corresponding button in exercise mode.
  2. Paste the URL of the Google Drive folder containing the .ent files
  3. Select a .ent file from the bank.
  4. Choose üìö Load as exercise. ENT creates an empty tree for the student and saves the file's tree as the model.

Comparison with the model

When you finish analyzing the exercise, use Compare. ENT will display two panels: Your analysis, on the left, and Model analysis, on the right. Differences are marked visually: correctly identified constituents and labels appear in green; incorrect ones in red.

At the top of the comparison view you can see the number of correct answers and the achievement percentage in relation to the model

From that view you can return to editing (‚Üê Back to editing) or start another exercise (New exercise).

8. Example bank (Google Drive)

Allows instructors to create a library of exercises accessible from the cloud.

If you don't have your own folder, ENT offers a link to the ENT example bank, a test folder with sample exercises that you can use to familiarize yourself with the system.

Minimum requirements

  • Bank files must be saved as .ent.
  • The Drive folder must be shared as "Anyone with the link" (read permission).
  • In ENT, simply paste the link (or ID) of the folder and connect.

How it is organized and what is shown

  • ENT lists the .ent files that are in the root of the folder.
  • It also checks the direct subfolders (one level) and lists their .ent files.
  • If a subfolder is not accessible or doesn't contain .ent files, it simply doesn't appear in the bank.

10. Exporting results

ENT exports the analysis as a PNG image with a transparent background so you can insert it into documents, slides, or articles.

  • Export diagram: exports the diagram as it appears (with boxes).
  • Export tree diagram: exports a "cleaner" style (technical mode), hiding editing controls and reducing non-essential visual elements.

During export, ENT automatically hides interactive buttons so they don't appear in the final image.
